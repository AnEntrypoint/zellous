
# ISOLATE-GLOBALS-PHASE2 EXECUTION PLAN

## Task Scope
Migrate 55 modules to export to window.__zellous instead of window.
Keep backward compat layer (window.lk → window.__zellous.voice, etc.) for safe rollback.

## Migration Strategy: Sectioned Waves

Instead of all 55 at once, migrate by subsystem in 6 waves. Each wave:
1. Modify 8-10 related modules to export to window.__zellous
2. Update backward compat layer if needed
3. Quick smoke test: load app, no console errors
4. Commit wave
5. Proceed to next wave

### Wave 1: State & Config (5 modules) — CRITICAL PATH
- state.js: window.state/stateSignals/config → window.__zellous.state/stateSignals/config
  (BUT: Keep window.state = window.__zellous.state for direct access, no getter needed for state)
- REASON: State is read/written by 50+ functions. Use direct assignment not getter for performance.
- MUTATION: Update backward compat to use assignment not defineProperty for state

### Wave 2: Voice Subsystem (7 modules) — CORE FEATURE
- nostr-voice.js → window.__zellous.voice
- nostr-voice-rtc.js → window.__zellous.voiceRtc
- nostr-voice-sfu.js → window.__zellous.voiceSfu
- nostr-voice-camera.js → window.__zellous.voiceCamera
- nostr-fsm.js → window.__zellous.fsm
- ptt.js → window.__zellous.ptt, .deafen, .vad
- Plus backward compat getters

### Wave 3: Network & Auth (4 modules)
- nostr-network.js → window.__zellous.net

- nostr-bans.js → window.__zellous.bans

### Wave 4: Server/Channel Management (6 modules)

### Wave 5: Chat & Messaging (4 modules)

## Per-Wave Procedure

For each module:
1. Find the line: `window.xxx = ...`
2. Change to: `window.__zellous.xxx = ...`
3. If module is also exported: `window.xxx = window.__zellous.xxx` (backward compat)
4. Check for special cases (see below)

## Special Cases

### state.js (Wave 1)
Current:
```
window.state = stateProxy;
window.stateSignals = state;
window.config = config;
```

Should become:
```
window.__zellous.state = stateProxy;
window.__zellous.stateSignals = state;
window.__zellous.config = config;
// Direct assignment for backward compat (not getter, for perf):
window.state = window.__zellous.state;
window.stateSignals = window.__zellous.stateSignals;
window.config = window.__zellous.config;
```

### nostr-fsm.js (Wave 2)
Current:
```
window.nostrFsm = { voiceMachine, peerMachine, cameraMachine };
```

Should become:
```
window.__zellous.fsm = { voiceMachine, peerMachine, cameraMachine };
window.nostrFsm = window.__zellous.fsm; // backward compat
```

### icons.js (Wave 6)
Current:
```
window.icons = icons;
window.getIcon = getIcon;
```

Should become:
```
window.__zellous.helpers = window.__zellous.helpers || {};
window.__zellous.helpers.icons = icons;
window.__zellous.helpers.getIcon = getIcon;
window.icons = icons;
window.getIcon = getIcon;
```

### ui-voice.js (Wave 6)
Current:
```
window.uiVoice = uiVoice;
window.uiMembers = uiMembers;
```

Should become:
```
window.__zellous.uiVoice = uiVoice;
window.__zellous.uiMembers = uiMembers;
window.uiVoice = uiVoice;
window.uiMembers = uiMembers;
```

## Testing Per Wave

After each wave, verify:
1. App loads (no errors in console)
2. window.__zellous.xxx is populated (spot check one module)
3. window.xxx still works (backward compat) (spot check one alias)
4. No network errors or relay disconnects

## Acceptance Criteria

✓ All 55 modules export to window.__zellous
✓ Backward compat layer: window.xxx = window.__zellous.xxx or getter
✓ App loads without console errors
✓ Quick smoke test: can join voice, send chat message
✓ No module loads before window.__zellous initialized
✓ Commit message reflects wave number and modules migrated

## Edge Cases & Unknowns

UNKNOWN: Do any modules check window.xxx in module init (before exports set)?
- Resolution: Scan each module for window.xxx references in top-level code
- If found: Move window.xxx assignment to end of module

UNKNOWN: Does state proxy rely on reference identity?
- Resolution: Verify state can be accessed as window.__zellous.state in all call sites
- If getter breaks reactivity: Use direct assignment window.state = window.__zellous.state

UNKNOWN: Can 6 waves be done sequentially in <2 hours?
- Depends on: # of call sites to update per module
- If >15 sites per module: May need to phase 3 (cross-module refs) before phase 2 finishes
- Conservative approach: Complete phase 2 (all exports), then phase 3 (all refs) together

## Decision: Sequential or Parallel Waves?

Recommendation: SEQUENTIAL within phase2 (do wave 1, test, commit, wave 2, etc.)
Reason: Each wave depends on previous working; catch issues early; cleaner commits.

Exception: Can parallelize waves if they have zero interdependencies:
- Wave 1 (state) is critical; must complete first
- Wave 2 (voice) and Wave 3 (network) could theoretically parallelize, but risky
- Keep sequential for safety
