## Migration Strategy

Phase 1: Create namespace container (no changes to functionality)
- Add window.__zellous = {} initialization script before module loads
- All modules still set window.xxx for now (backward compat layer)
- window.__zellous.xxx = window.xxx mapping done at module init end

Phase 2: Update module exports (safe to parallelize)
- Each module appends its exports to window.__zellous instead of window
- Keep window.xxx = window.__zellous.xxx for backward compatibility
- Total: 55 module modifications, can batch by subsystem

Phase 3: Update all references (complex, must be sequential)
- Search/replace all internal cross-module calls
- ui-actions.switchChannel() calls lk.connect() → window.__zellous.voice.connect()
- nostr-channels.js references window.ui → window.__zellous.ui
- etc. (estimated 100+ call sites)

Phase 4: Test backward compatibility layer
- Verify window.lk still works (points to window.__zellous.voice)
- Verify window.state still works
- Full integration test with existing app

Phase 5: Remove backward compatibility (if user approves)
- Delete all window.xxx = ... assignments
- Delete backward compat layer
- Update CLAUDE.md

## RISK ASSESSMENT

High Risk: State management. state.js sets window.state = stateProxy. All modules read/write window.state directly. Need to verify state proxy still works through getter.

Medium Risk: Moderation and bans logic depends on checking window.serverRoles, window.moderation. If accessed before namespace init, breaks.

Low Risk: Most modules are self-contained (voice, media, etc.).

## Decision Point

Should state, stateSignals, config remain at window level (not namespaced)?

Rationale for staying at window level:
- State is accessed by 50+ functions directly
- Signals pattern (Preact) relies on reference identity
- Getter forwarding (window.__zellous.state → window.state) adds indirection

Rationale for namespacing:
- Prevents parent page collision with state object
- Cleaner API surface
- Embedding use case requires it

DECISION: Namespace state under window.__zellous, but use getter pattern to avoid indirection. window.__zellous.state is a live reference, not a copy.
